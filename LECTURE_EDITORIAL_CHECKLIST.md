# Lecture Editorial Checklist

This checklist freezes the current **Lecture 2** standard into an explicit acceptance gate for every other lecture module in the course.

## Core rule

No lecture is considered complete until it can reasonably replace class attendance for a student who needs to learn the lecture from the module itself.

## Required acceptance items

Every completed lecture must include all of the following:

1. A clear **"why this lecture matters"** opening.
2. A module-local glossary that teaches notation in context rather than as a flat lookup list.
3. A full theory chain instead of compressed note fragments.
4. At least one fully explained derivation or mechanism chain.
5. Explicit benchmark logic where relevant:
   - what changes in this lecture
   - what stays from the benchmark
   - what students must carry forward
6. One guided practice problem that rehearses the lecture's actual reasoning demands.
7. One 5-question concept-heavy quiz with useful explanations.
8. Source-grounded tutor support through citations and rich enough section content to support useful answers.
9. One figure-backed interpretation section whenever the authoritative source pack contains a figure that materially improves understanding.
10. At least one explicit **common-mistake or exam-trap block**.

## Editorial expectations

Each lecture should also satisfy these style and pedagogy rules:

- The lecture must read as part of a **connected course**, not as an isolated essay.
- The opening, setup, compact takeaway, and recap should all help students understand the course red thread.
- Exams and problem sets should be used to infer:
  - mastery requirements
  - likely confusion points
  - the reasoning patterns students need to build across the course
- Exams and problem sets should shape emphasis, sequencing, depth, intuition, and reinforcement across the whole lecture, not only a final exam-prep section.
- Math must render in LaTeX/KaTeX form with no raw underscore notation shown to students.
- Figures should default to tightly cropped screenshots from authoritative PDFs unless a redraw is clearly better for learning.

## Practical acceptance test

Before shipping a lecture, ask:

- Can a student who missed class read this module from start to finish and still understand the benchmark, the new mechanism, the dynamics, the interpretation, and the carry-forward takeaway?
- Does the lecture teach the symbols, assumptions, and mechanism slowly enough to support later lectures and exam reasoning?
- Is there at least one place where the lecture explicitly warns the student about a likely mistake or misleading shortcut?
- If the source pack contains a helpful visual, is that visual actually present in the module with a caption that explains what to notice?

If the answer to any of these is **no**, the lecture is not finished.
